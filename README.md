## 📋 Description

**NetPractice** is a practical networking exercise from the 42 curriculum. The goal is to configure **10 broken network diagrams** by assigning correct IP addresses, subnet masks, default gateways, and routing table entries so that each network functions properly.

The project covers the fundamentals of **TCP/IP addressing**:

- 🔢 How to read and assign IP addresses and subnet masks
- 🔀 How routers forward packets between different subnets
- 🗺️ How to configure routing tables (destination, next hop, interface)
- 🚪 How the default gateway works as the exit point of a local network

> All networks are **simulated** and run locally in the browser — no real infrastructure required.

## 🚀 Instructions

### ▶️ Launching the training interface

**Option 1 — using the provided script:**
```bash
./run.sh
```
This starts a local web server and opens the interface in your default browser.

**Option 2 — manual launch (if `run.sh` fails):**
```bash
python3 -m http.server 49242
```
Then open your browser at: **`http://localhost:49242`**

> 💡 You may change the port number if 49242 is already in use.

### 🧠 Networking concepts studied

| Concept | Description |
|---|---|
| 🔢 **TCP/IP addressing** | IPv4 structure, 4 octets, decimal notation |
| 🎭 **Subnet masks** | Separating network/host portions, CIDR notation |
| 📐 **CIDR notation** | `/24`, `/27`, `/30`... block sizes and usable ranges |
| 🚪 **Default gateway** | Router interface used to exit a local network |
| 🗺️ **Routing tables** | Destination, next hop, interface |
| 🔀 **Routers** | Multi-interface devices connecting different networks |
| 🔌 **Switches** | Layer 2 devices, transparent to IP routing |
| 🏗️ **OSI model** | Layer 3 focus — IP addressing and routing |
| 🔒 **Private address ranges** | RFC 1918: `10.x`, `172.16–31.x`, `192.168.x` |
| 🚫 **Broadcast & network addresses** | First/last addresses of a subnet — not assignable |
